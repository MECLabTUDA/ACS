Kerberos ist ein Sicherheitssystem um die Zugriffsdauer von Useren zu begrenzen. Dabei wird beim einloggen (local und remote) ein ticket ausgesetllt welches eine Laufzeit von 10 Stunden hat.
Diese kann durch den Befehl 'krenew' um eine gewisse Zeitspanne verlängernt werden. Danach muss man mit 'kinit' ein neues Ticket angefragt werden.

Ein ticket gilt jedoch nur für die aktuelle shell. Öffnet man zum Beispiel über tmux eine weitere, aktualisiert das Ticket aber nur für die ursprüngliche shell, läuft das Ticket nach 10 Stunden in der tmux session aus.
Da man für längere Jobs eh immer tmux oder screen nutzt, hier deswegen eine Anleitung wie man das Problem löst (local wie remote):

1. pip install pexpect
2. In das kinit_auto.py script username und password eintragen
3. tmux session erstellen (tmux new)
4. Neue tmux pane öffnen (Ctrl+B ")
5. Neue pane aktivieren (Ctrl+B DOWN)
6. kinit_auto.py alle 5 Stunden ausführen (watch -n 18000 kinit_auto.py)
7. Optional kann man dann die pane wieder verstecken

Jobs in der tmux session läuft dann endlos ohne unterbrochen zu werden.
Man kann das ganze sicher noch verfeinern (password nicht in plaintext, oder das Schritte 3-7 automatisiert werden).
